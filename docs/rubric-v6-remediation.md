# Rubric v6修复说明

## 状态

`securities-expense-audit-formal-15-v8`与`atomic-binary-checklist-v6`已替代v7/v5，原GATE4/GATE5运行和排名保留为历史诊断材料，不得直接转换成v6排名。新版本修改了正式题面、检查项和评分规则，必须重新执行GATE3冻结、GATE4运行和GATE5判卷。

## 已修复问题

| 评审问题 | v6修复 |
| --- | --- |
| 正常题和陷阱题泄露结论 | 六道单案题统一为中性句式，只提供一个线索记录，不出现异常类型、“若不构成异常”或核查维度提示 |
| 全年题奖励扫射式作答 | 异常发现和`record_id`分别设置50%、80%、100%精确率与召回率阈值，漏报和多报同量级扣分 |
| 陷阱题同一事实重复计分 | 删除全部`reverse-check-*`，每题只保留一个要求工具轨迹或工作产物的独立核验过程项 |
| L3-009检查项过度展开 | 从73项压缩为27项；所有题限制为不超过30项 |
| 确定性项目只能向上申诉 | 集合、数量、阈值和Schema项目由规则最终裁决，不再提交Judge复核 |
| 严重误报与普通漏写等价 | 无异常题误报封顶40分；批量题记录或发现精确率低于50%封顶50分 |
| Judge方差未知 | 固定抽样20%的语义项做第二次判卷，分歧时第三次裁决；输出一致率及区间 |
| 增强标准构造上不可达 | 改为Q至少提升5分、至少两个有改善余量的题族提升1分，且任何关键题族下降不超过5分 |
| 旧结果与新规则可能混用 | v6使用独立的`gate5_grades_v6.jsonl`、`gate5_judge_v6/`和`output/gate5_v6/`路径 |

## 重新评测前置条件

1. 使用开发题对六个框架的中性单案题路由和批量精确率/召回率提交进行Canary验证。
2. 重新生成GATE3冻结锁，不沿用v5锁文件。
3. 四个语义项批次照常判卷；固定抽样复判必须达到90%一致率，否则排名只报告同层级，不给严格先后。
4. 正式结果必须同时带有`dataset_id`和`rubric_version`，缺少版本标识的旧成绩不得进入v6汇总。
