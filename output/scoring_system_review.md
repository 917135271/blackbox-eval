# 评分系统复核与修订

## 结论

原评分公式存在重复计分：Q已经来自逐题Rubric，而Rubric中包含record_ids、制度依据、误报控制和提交格式；E又重复计算记录、制度和陷阱题，S又重复统计提交成功。原Token缺失值按硬上限计分，也会把遥测缺失误当成高消耗。

修订后，逐题Rubric是业务质量的唯一评分来源，证据指标仅用于解释得失，不再二次进入总分。

评分版本标识为`gate5-scoring-v2`，用于阻止旧公式结果和新公式结果混用。

## 新评分公式

$Total = 0.75Q + 0.15S + 0.10F$

- Q：逐题Rubric剔除统一提交和纯格式项后归一化的业务质量平均分。
- S：在时限内正常运行、成功提交且通过Schema的任务比例。
- F：按单题中位耗时计算的时效得分。

## 质量分解

以下指标不进入Total，只用于解释Q：

- record_id宏平均F1；
- 制度类Rubric评分点得分率；
- 误报控制类Rubric评分点得分率；
- 陷阱题特异度。

Token和API用量继续记录。在六个框架都能使用相同缓存口径完整上报前，不参与排名。

## 兼容边界

新公式只适用于`securities-expense-audit-formal-15-v6`和`atomic-binary-checklist-v4`。三道无异常陷阱题的核心结论与空异常输出属于题目通过条件，不改变等权原始分，但任一失败都会将该题标记为未通过。旧报告中的E和Total不能与新结果直接比较；旧逐题Rubric判分仍可保留用于轨迹分析，但排名必须按新公式重新汇总。
