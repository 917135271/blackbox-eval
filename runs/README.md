# Run Artifact Retention

Formal run directories keep the evidence needed to review and reproduce an
evaluation: run manifests, final submissions, structured Hook events, tool
calls, subagent records, task checkpoints, workspace artifacts, Judge outputs,
and aggregate reports.

New `artifacts/trajectory.jsonl` files are not committed. Some frameworks write
the full cumulative model context on every event, which can produce a single
task file hundreds of megabytes in size without adding equivalent audit value.
The structured logs listed above are the canonical Git evidence. Raw cumulative
transcripts can be regenerated by rerunning the frozen runner configuration.
